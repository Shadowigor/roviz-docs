Detailed Description

Class DomHelper acts as a helper class for storing and retrieving data to or from a DOM document.

At the moment it helps you with storing and retrieving QVariant data and QObject properties.

◆ saveVariant() [1/2]

QDomElement DomHelper::saveVariant ( const QVariant &  variant,
const QString  tagName,
QDomDocument &  doc,
const QString &  name = QString::null 
)
static

Creates a new DOM element tagName, saves variant to it and returns the created element.

The created element will be tagged tagName and has at least one attribute named DomHelper::TypeTag with the variant's type name.

If name is not empty, another attribute named DomHelper::NameTag with value name will be created and added to the created DOM element.

If variant has converters to and from string, i.e. 

variant.canConvert(QMetaType::QString) && QVariant(QString()).canConvert(variant.userType())

is true, an attribute named DomHelper::ValueTag with the variant's string value will be added to the created DOM element.

Some variant type's (e.g. QVariant::List, QVariant::StringList) value will be added as human readable children to the created element.

If the variant value cannot be handled in these ways, the variant will be serialized, converted to base64 and added as text node to the created DOM element.

Parameters
variantThe variant to be saved.
tagNameThe tag name of the created DOM element.
docUsed to create child nodes.
nameAn optional name. If not empty, an attribute with name DomHelper::NameTag and value name will be added to the created DOM element.
Returns
A DOM element with tag name tagName which has variant stored to it or a null element if variant cannot be saved.
